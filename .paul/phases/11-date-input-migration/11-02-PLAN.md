<task type="auto">
  <name>Task 1: Create DateTimePicker shared component</name>
  <files>src/shared/ui/form/DateTimePicker.tsx, src/shared/ui/form/index.ts</files>
  <action>
    Create `src/shared/ui/form/DateTimePicker.tsx` — a React Hook Form Controller wrapper that combines DatePicker (calendar popover) + HTML time input:

    **Component design:**
    - Props interface `DateTimePickerFormProps` with: control, name, label, error, required, disabled, placeholder (same pattern as DatePicker.tsx)
    - Uses `<Controller>` from react-hook-form (same pattern as DatePicker.tsx)
    - Layout: horizontal flex row — DatePicker popover button (flex-1) + time input (w-[120px])
    - Date part: reuses Calendar component inside Popover (same as DatePicker.tsx)
    - Time part: native `<input type="time">` styled with Shadcn Input classes
    - Returns "YYYY-MM-DDTHH:mm" string — same format as datetime-local
    - Parses incoming value: "YYYY-MM-DDTHH:mm" → split into date ("YYYY-MM-DD") + time ("HH:mm")
    - Handles empty value: empty string → date undefined, time empty string
    - **CRITICAL:** Incomplete datetime handling — BOTH date AND time must be set for a valid value
      - When date selected (no time yet): field.onChange("") — incomplete datetime, do NOT default to 00:00
      - When time changed (no date yet): field.onChange("") — incomplete datetime
      - When both date AND time set: field.onChange("YYYY-MM-DDTHH:mm") — complete datetime only
    - Vietnamese locale for calendar via date-fns `vi`
    - Label + required asterisk + error message display (same pattern as DatePicker.tsx)
    - Uses relative imports within shared folder (same as DatePicker.tsx)
    - Time input: step="60" for minute-level precision (no seconds needed)
    - Accessibility: time input gets aria-label="Time selection" for screen readers

    **Export:**
    - Add `DateTimePicker` and `DateTimePickerFormProps` to `src/shared/ui/form/index.ts`

    **Avoid:**
    - Do NOT import from `@/shared/...` paths — use relative imports (composite project constraint from 11-01)
    - Do NOT return undefined — always return empty string for null/undefined
    - Do NOT create a separate time picker component — inline the time input within DateTimePicker
  </action>
  <verify>npm run build completes with zero errors; verify time input keyboard navigation works (Tab to focus, arrows to adjust)</verify>
  <done>AC-1 satisfied: DateTimePicker renders DatePicker + time input, returns YYYY-MM-DDTHH:mm only when both parts set</done>
</task>
