<task type="auto">
  <name>Task 2: Create DatePicker wrapper with React Hook Form Controller</name>
  <files>src/shared/ui/form/DatePicker.tsx, src/shared/ui/form/index.ts</files>
  <action>
    Create a DatePicker wrapper component that:
    1. Exports a named export `DatePicker` from `src/shared/ui/form/DatePicker.tsx`
    2. Uses React Hook Form's `Controller` pattern for form integration
    3. Accepts props: `control`, `name`, `label`, `error`, `required`, `disabled`
    4. Renders Shadcn DatePicker with proper styling
    5. Integrates with dayjs for date handling (reuse existing dayjs(vi) setup)
    6. Returns ISO string (YYYY-MM-DD) for form value consistency — **date-only, timezone-agnostic**
    7. Shows validation error message when `error` prop is provided
    8. **Handles null/undefined values by returning empty string** — prevents undefined crashes in form submission
    9. **Exports explicit TypeScript type signature** for public API consumption

    Follow the existing FormSection component pattern for consistent styling.

    Avoid: Using native Date object directly — convert to/from dayjs for locale support
    Avoid: Returning undefined — always return string (empty or ISO date)
  </action>
  <verify>
    Run: `grep -l "DatePicker" src/shared/ui/form/index.ts` — should export DatePicker
    Check: Component uses Controller from react-hook-form
    Check: Component handles Vietnamese locale via dayjs(vi)
  </verify>
  <done>
    AC-2 satisfied: DatePicker wrapper integrates with React Hook Form Controller
  </done>
</task>

<task type="auto">
  <name>Task 3: Create DateRangePicker wrapper for list page filters</name>
  <files>src/shared/ui/form/DateRangePicker.tsx, src/shared/ui/form/index.ts</files>
  <action>
    Create a DateRangePicker wrapper component that:
    1. Exports a named export `DateRangePicker` from `src/shared/ui/form/DateRangePicker.tsx`
    2. Does NOT use React Hook Form Controller — meant for uncontrolled filter inputs
    3. Accepts props: `value`, `onChange`, `placeholder`, `fromPlaceholder`, `toPlaceholder`, `disabled`, `invalidRangeMessage`
    4. Renders two DatePicker instances (From/To) in a flex row
    5. Returns/accepts `{ from: Date | undefined; to: Date | undefined }` object
    6. Uses Vietnamese locale via existing dayjs(vi) setup
    7. **Validates that from ≤ to, displays `invalidRangeMessage` when invalid**
    8. **Exports explicit TypeScript type signature** for public API consumption

    Follow spacing: gap-2 between the two date pickers.

    Avoid: Adding form validation — this is for list page filters, not form submission
    Avoid: Allowing invalid ranges (from > to) to propagate silently
  </action>
  <verify>
    Run: `grep -l "DateRangePicker" src/shared/ui/form/index.ts` — should export DateRangePicker
    Check: Component does not use Controller pattern
    Check: Component accepts and returns DateRange object
  </verify>
  <done>
    AC-3 satisfied: DateRangePicker wrapper handles from/to dates without form integration
