<objective>
## Goal
Create the trip schema with `trips` and `trip_staff` tables, including proper FK references to routes/vehicles/employees, a partial unique index enforcing max 1 driver per trip, and realistic Vietnamese seed data.

## Purpose
Trips are the operational core of the system — they connect routes, vehicles, and staff into scheduled departures. This schema enables Phase 6 (Trip Scheduling) to build conflict detection, calendar views, and driver schedule management on top of a solid data foundation.

- `supabase/migrations/20260411150001_trip_triggers.sql` — updated_at triggers for both tables
- `supabase/seed.sql` — appended trip seed data (6 trips, 9 trip_staff assignments)
</objective>

<task type="auto">
  <name>Task 1: Trip Schema Migration</name>
  <files>supabase/migrations/20260411150000_trip_schema.sql</files>
  <action>
    Create migration file following established patterns (see fleet_schema.sql and route_schema.sql):

    **trips table:**
    - id: uuid PK default gen_random_uuid()
    - route_id: uuid NOT NULL, FK → routes(id) ON DELETE RESTRICT
    - vehicle_id: uuid NOT NULL, FK → vehicles(id) ON DELETE RESTRICT
    - departure_time: timestamptz NOT NULL
    - estimated_arrival_time: timestamptz NOT NULL
    - actual_arrival_time: timestamptz (nullable — filled on trip completion)
    - status: text NOT NULL DEFAULT 'scheduled' CHECK (status IN ('scheduled', 'in_progress', 'completed', 'cancelled'))
    - price_override: numeric(12,2) (nullable — when set, overrides route.base_price)
    - notes: text
    - created_at, updated_at: timestamptz NOT NULL DEFAULT now()
    - CHECK constraint: departure_time < estimated_arrival_time
    - CHECK constraint: price_override >= 0 (only when not null)
    - CHECK constraint: actual_arrival_time IS NULL OR actual_arrival_time > departure_time

    **Indexes on trips:**
    - idx_trips_route_id on route_id
    - idx_trips_vehicle_departure on (vehicle_id, departure_time) — composite index for "what trips does vehicle X have around time Y?" queries. Replaces separate vehicle_id index. <!-- audit-added -->
    - idx_trips_status on status
    - idx_trips_departure_time on departure_time (keep for standalone time-range queries)

    **trip_staff table:**
    - trip_id: uuid NOT NULL, FK → trips(id) ON DELETE CASCADE
    - employee_id: uuid NOT NULL, FK → employees(id) ON DELETE RESTRICT
    - role: text NOT NULL CHECK (role IN ('driver', 'assistant'))
    - notes: text
    - created_at, updated_at: timestamptz NOT NULL DEFAULT now()
    - PRIMARY KEY (trip_id, employee_id) — composite PK, following junction table pattern

    **Indexes on trip_staff:**
    - idx_trip_staff_employee_id on employee_id
    - idx_trip_staff_role on role
    - Unique partial index: CREATE UNIQUE INDEX idx_trip_staff_one_driver ON trip_staff (trip_id) WHERE role = 'driver'
      This enforces max 1 driver per trip at the database level.

    **Comments:** Add COMMENT ON TABLE and COMMENT ON COLUMN for all tables/columns, following established pattern. Also add COMMENT ON INDEX for the partial unique index explaining the business rule:
    ```sql
    COMMENT ON INDEX idx_trip_staff_one_driver IS 'Business rule: each trip may have at most one driver. This partial unique index prevents assigning multiple drivers to the same trip.';
    ``` <!-- audit-added -->

    Avoid: Do NOT add application-level conflict detection (overlapping schedules) — that's plan 02-07 scope.
  </action>
