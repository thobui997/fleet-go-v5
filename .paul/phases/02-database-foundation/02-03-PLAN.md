<objective>
## Goal
Create the route database schema: stations, routes, and route_stops tables with proper constraints, indexes, triggers, and realistic Vietnamese seed data.

## Purpose
Routes and stations are the backbone of trip scheduling (Phase 6) and ticketing (Phase 7). This schema must support:
- Station management (CRUD with location data)
- Route definition with origin/destination, distance, duration, base price
- Ordered intermediate stops via route_stops junction table
- Price and schedule calculations between any two stops on a route

## Output
- 2 migration files: route schema + route triggers
- Updated seed.sql with stations, routes, and route_stops data
</objective>

<task type="auto">
  <name>Task 1: Route Schema Migration</name>
  <files>supabase/migrations/20260411140000_route_schema.sql</files>
  <action>
Create route schema migration with 3 tables following established patterns from 02-01 and 02-02.

**stations table:**
- id uuid PK (gen_random_uuid())
- name text NOT NULL UNIQUE CHECK (trim(name) <> '') — station display name <!-- audit-added: empty string protection -->
- code text UNIQUE — short code for station (e.g., "HN-GB")
- address text — street address
- city text NOT NULL — city name
- province text — province/region
- latitude numeric(9,6) CHECK (latitude BETWEEN -90 AND 90) — GPS latitude <!-- audit-added: coordinate range validation -->
- longitude numeric(9,6) CHECK (longitude BETWEEN -180 AND 180) — GPS longitude <!-- audit-added: coordinate range validation -->
- is_active boolean NOT NULL DEFAULT true
- created_at, updated_at timestamptz

Indexes: idx_stations_city (city), idx_stations_is_active (is_active), idx_stations_province (province)

Comments on table and all columns (follow pattern from fleet_schema.sql).

**routes table:**
- id uuid PK (gen_random_uuid())
- name text NOT NULL UNIQUE CHECK (trim(name) <> '') — route display name (e.g., "Ha Noi → Da Nang") <!-- audit-added: empty string protection -->
- origin_station_id uuid NOT NULL FK → stations(id) ON DELETE RESTRICT
- destination_station_id uuid NOT NULL FK → stations(id) ON DELETE RESTRICT
- distance_km numeric(8,2) NOT NULL CHECK (distance_km > 0)
- estimated_duration interval NOT NULL CHECK (estimated_duration > '0'::interval)
- base_price numeric(12,2) NOT NULL CHECK (base_price >= 0)
- is_active boolean NOT NULL DEFAULT true
- created_at, updated_at timestamptz
- CONSTRAINT different_origin_destination CHECK (origin_station_id != destination_station_id)

Indexes: idx_routes_origin_station_id, idx_routes_destination_station_id, idx_routes_is_active

Comments on table and all columns.

**route_stops table (junction with ordering):**
- route_id uuid NOT NULL FK → routes(id) ON DELETE CASCADE
- station_id uuid NOT NULL FK → stations(id) ON DELETE RESTRICT
- stop_order int NOT NULL CHECK (stop_order > 0)
- estimated_arrival interval CHECK (estimated_arrival > '0'::interval) — nullable, time offset from departure
- pickup_allowed boolean NOT NULL DEFAULT true
- dropoff_allowed boolean NOT NULL DEFAULT true
- PRIMARY KEY (route_id, station_id) — composite PK following user_roles pattern
- UNIQUE (route_id, stop_order) — prevents two stops at same position

Indexes: idx_route_stops_station_id

Comments on table and all columns.

