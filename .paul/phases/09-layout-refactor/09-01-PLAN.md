<objective>
## Goal
Refactor `AppLayout` and `DataTable` to establish the viewport-constraint foundation: the app shell never scrolls as a whole, and DataTable automatically provides a sticky-header scrollable body within whatever height its container offers.

## Purpose
Currently `<main>` uses `overflow-y-auto` which allows the entire page to scroll freely. `DataTable` has no height constraints and no sticky header. Fixing these two shared components unlocks all list pages to adopt viewport-constrained layouts without duplicating logic per page.

## Output
- `app-layout.tsx`: root is `h-screen overflow-hidden`; content column is `flex flex-col h-screen`; `<main>` is `flex-1 overflow-hidden p-6 flex flex-col`
- `data-table.tsx`: root is `flex flex-col h-full gap-4`; table wrapper is `flex-1 min-h-[200px] overflow-auto rounded-md border`; `<TableHeader>` rows are `sticky top-0 bg-card z-10`; pagination is `flex-none`
</objective>

    1. Root div: change `min-h-screen bg-background` → `h-screen overflow-hidden bg-background`

       - `overflow-hidden` prevents the root from growing beyond viewport

    2. Content column div (the one with `md:ml-64` / `md:ml-16`): add `h-screen flex flex-col`
       - This makes the content column fill the full viewport height
       - `flex flex-col` enables Header + main to stack vertically with proper sizing

       - `flex-1`: takes all remaining height after the fixed-height Header
       - `overflow-hidden`: clips content — individual pages control their own scroll
       - `p-6`: keep padding here (pages will no longer carry their own p-6 after 09-02)
       - `flex flex-col`: allows page root with `h-full` to fill the padded content area

    Do NOT touch Sidebar, Header, mobile overlay logic, scroll-lock effects, or Escape key handler.
    Do NOT add `overflow-y-auto` anywhere in this file.

<task type="auto">
  <name>Task 2: Refactor DataTable for sticky header and constrained scroll</name>
  <files>src/shared/ui/data-table.tsx</files>
  <action>
    Make the following targeted changes to the `DataTable` component:

    1. Root wrapper div: change `className="space-y-4"` → `className="flex flex-col h-full gap-4"`
       - `h-full`: fills whatever height the parent offers
       - `flex flex-col`: stacks table wrapper and pagination vertically
       - `gap-4`: replaces `space-y-4` (correct flex equivalent, visually identical 16px gap)

    2. Table outer wrapper div: change `className="rounded-md border"` →
       `className="flex-1 min-h-[200px] overflow-auto rounded-md border"`
       - `flex-1`: takes all available height (between top of DataTable and pagination)
       - `min-h-[200px]`: minimum usable scroll area even on very short viewports
       - `overflow-auto`: the ONLY scroll container — rows scroll here, not the page

    3. `<TableHeader>` component: add `className="sticky top-0 z-10 bg-card"`
       - `sticky top-0`: pins header at top of the scroll container
       - `bg-card`: opaque background matching the table card — prevents row bleed-through
       - `z-10`: renders header above table rows; Radix DropdownMenus are portaled to body (z-50+) so no conflict

    4. Pagination wrapper div (the `flex items-center justify-between px-2` div):
       add `flex-none` to the className — pins it below the scrollable table.

    Do NOT change any logic, sorting, loading states, skeleton rows, empty state, or pagination controls.
    Do NOT change ColumnDef, DataTableProps, or DataTablePagination interfaces.
