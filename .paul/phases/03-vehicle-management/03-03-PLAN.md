## Database Schema
Table: `maintenance_logs`
| Column | Type | Constraints |
|--------|------|-------------|
| id | uuid | PK, DEFAULT gen_random_uuid() |
| vehicle_id | uuid | NOT NULL, FK → vehicles(id) ON DELETE CASCADE |
| type | text | NOT NULL, CHECK IN ('routine','repair','inspection','emergency') |
| description | text | NOT NULL |
| cost | numeric(12,2) | DEFAULT 0, CHECK >= 0 |
| performed_by | text | NULLABLE |
| performed_at | date | NOT NULL, DEFAULT current_date |
| next_due_date | date | NULLABLE |
| odometer_reading | int | NULLABLE, CHECK >= 0 |
| notes | text | NULLABLE |
| created_at | timestamptz | NOT NULL, DEFAULT now() |
| updated_at | timestamptz | NOT NULL, DEFAULT now() (trigger-managed) |

Indexes: vehicle_id, type, performed_at, next_due_date
RLS: maintenance:read for SELECT, maintenance:write for INSERT/UPDATE, is_admin() for DELETE
FK: vehicle_id CASCADE on delete

    - `MaintenanceLog` interface matching all DB columns with explicit TypeScript types:
      - id: string; vehicle_id: string; type: MaintenanceType; description: string
      - cost: number (NOT string — Supabase returns numeric(12,2) as JS number; DataTable uses toLocaleString which requires number) <!-- audit-added: S7 -->
      - performed_by: string | null; performed_at: string; next_due_date: string | null
      - odometer_reading: number | null; notes: string | null
