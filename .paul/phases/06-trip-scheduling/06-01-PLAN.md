    Define `serializeToInsert(values: TripFormValues): TripInsert`: <!-- audit-added: fixed typo TripImport → TripInsert -->
    - Convert datetime-local string to ISO: `new Date(values.departure_time).toISOString()`

    Define `toDatetimeLocal(iso: string): string`:
    - **IMPORTANT**: Do NOT use `iso.slice(0, 16)` — this extracts UTC time digits, not local time. <!-- audit-added: timezone correctness fix -->
    - Correct implementation: parse ISO, format in local timezone:
    ```typescript
    export function toDatetimeLocal(iso: string): string {
      const d = new Date(iso);
      const pad = (n: number) => String(n).padStart(2, '0');
      return `${d.getFullYear()}-${pad(d.getMonth() + 1)}-${pad(d.getDate())}T${pad(d.getHours())}:${pad(d.getMinutes())}`;
    }
    ```
    - Why: `iso.slice(0, 16)` on "2026-04-16T01:30:00Z" yields "2026-04-16T01:30" (UTC 01:30), but the user entered 08:30 Vietnam time. The datetime-local input interprets this as local 01:30 — silently corrupting the display.
