<objective>
## Goal
Build a monthly calendar view displaying trips on a date grid, and a personal "My Schedule" page showing the logged-in employee's assigned trips — replacing both placeholder routes.

## Purpose
Dispatchers need a visual overview of trips across days to spot gaps and overlaps. Drivers/assistants need a quick view of their own upcoming assignments. These views complete the Trip Scheduling phase and unlock the dashboard readiness for Phase 8.

## Output
- `fetchTripsByDateRange` API + hook in @entities/trip
- `fetchMySchedule` API + hook in @entities/trip-staff
- Trip Calendar page at `/trips/calendar` (monthly grid)
- My Schedule page at `/my-schedule` (assigned trip list)
- Both placeholder routes wired to real pages
</objective>

```gherkin
Given the calendar page at /trips/calendar
When the page loads
Then a monthly grid displays with day cells containing trip cards
And each trip card shows route name, departure time, status badge, and vehicle plate
And month navigation (prev/next/today) changes the visible month
And trips are fetched for the full visible month range
And today's date cell is visually highlighted (distinct background/ring)
```

## AC-3: My Schedule API
```gherkin
Given the trip-staff entity API
When fetchMySchedule is called with a Supabase auth user ID
Then it returns trip_staff records with trip details (route, vehicle, times, status)
For the employee matching that user_id
Ordered by departure_time ascending
And when no employee record exists for the user (PGRST116)
Then it returns an empty array without throwing an error
```

## AC-4: My Schedule Page
```gherkin

Then the page displays their assigned trips as a card/list view
And each card shows trip route, departure/arrival times, their role (driver/assistant), and trip status
And trips are split into "upcoming" and "past" sections
```

    - <!-- audit-added --> PostgREST `referencedTable` ordering is unreliable for nested joins. Do NOT rely on it. Instead, sort client-side after fetch: `results.sort((a, b) => new Date(a.trip.departure_time).getTime() - new Date(b.trip.departure_time).getTime())`
