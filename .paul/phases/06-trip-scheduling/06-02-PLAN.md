  <files>src/entities/trip-staff/model/types.ts, src/entities/trip-staff/api/trip-staff.api.ts, src/entities/trip-staff/api/trip-staff.queries.ts, src/entities/trip-staff/index.ts</files>
  <action>
    Create the trip-staff entity slice following the established FSD pattern (model/types → api/*.api → api/*.queries → index.ts):

    **types.ts:**
    - `StaffRole = 'driver' | 'assistant'`
    - `TripStaff`: trip_id, employee_id, role (StaffRole), notes (string|null), created_at, updated_at
    - `TripStaffWithDetails extends TripStaff`: add `employee: { id: string, profiles: { id: string, full_name: string | null } | null } | null`
    - `TripStaffInsert`: { trip_id: string, employee_id: string, role: StaffRole, notes?: string | null }
    - `StaffConflict`: { trip_id: string, role: StaffRole, trip: { id: string, departure_time: string, estimated_arrival_time: string, route: { name: string } | null } }

    **trip-staff.api.ts:**
    - `STAFF_SELECT = '*, employee:employees(id, profiles(id, full_name))'`
    - `fetchTripStaff(tripId: string)`: select from trip_staff with STAFF_SELECT where trip_id = tripId, order by role
    - `addTripStaff(input: TripStaffInsert)`: insert, return with STAFF_SELECT, single()
    - `removeTripStaff(tripId: string, employeeId: string)`: delete where trip_id AND employee_id match
    - `fetchStaffConflicts(employeeId, departureTime, arrivalTime, excludeTripId?)`:
      - Query trip_staff with select `'trip_id, role, trips(id, departure_time, estimated_arrival_time, status, route:routes(name))'` where employee_id matches
      - Client-side filter: status must be 'scheduled' or 'in_progress', trip_id !== excludeTripId
      - Time overlap: `new Date(departureTime) < new Date(trip.estimated_arrival_time)` AND `new Date(arrivalTime) > new Date(trip.departure_time)`
      - Return array of StaffConflict

    **trip-staff.queries.ts:**
    - `useTripStaff(tripId)`: query key `['trip-staff', tripId]`, enabled when !!tripId
    - `useAddTripStaff`: mutation, on success invalidate `['trip-staff']`
    - `useRemoveTripStaff`: mutation, on success invalidate `['trip-staff']`
    - `useStaffConflicts(employeeId, departureTime, arrivalTime, excludeTripId?)`: query key `['staff-conflicts', employeeId, departureTime, arrivalTime, excludeTripId]`, enabled when !!employeeId

    **index.ts:** Export all types and hooks

    Use `import { supabase } from '@shared/api'` pattern established in other entities.
  </action>
