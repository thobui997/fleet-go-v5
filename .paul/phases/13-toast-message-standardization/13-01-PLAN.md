<objective>
## Goal
Standardize all toast messages in dialog-based files (form dialogs + delete dialogs) to use a consistent 3-field structure with explicit title, description, and variant.

## Purpose
Toast messages currently have 5+ inconsistent patterns: some use only `description`, some use only `title`, some have no `variant` on success toasts, and error toasts sometimes omit `title: 'Lỗi'`. This makes the app feel unpolished and unprofessional. Phase 13 enforces a single standard across all 26 files that need changes (split across 2 plans; this plan covers dialogs).

## Output
17 dialog files updated. All toast calls follow the standard pattern defined below.
</objective>

<context>
## Project Context
@.paul/PROJECT.md

## Standard Toast Pattern (MUST FOLLOW)

Three patterns — pick based on outcome:

**Success:**
```tsx
toast({ title: 'Thành công', description: '[what happened in Vietnamese]', variant: 'success' });
```

**Error:**
```tsx
toast({ title: 'Lỗi', description: mapXxxError(...), variant: 'destructive' });
```

**Warning (partial success — employee saved but role failed, etc.):**
```tsx
toast({ title: 'Cảnh báo', description: '[what partially succeeded and what failed]', variant: 'destructive' });
```
