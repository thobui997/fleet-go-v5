## AC-1: Route form — create mode
```gherkin
Given a user on the routes list page (/routes)

Then they navigate to /routes/new
```

## AC-2: Route form — edit mode
```gherkin
Given a route exists with id "abc"
When the user clicks "Sửa" on that route row
Then they navigate to /routes/abc/edit
And the form is pre-filled with data fetched via useRoute(id)

```

```gherkin
And the user can add/remove/reorder intermediate stops
```

Create `route-form-page.tsx` following the established full-page form pattern (see TripFormPage reference). Key implementation:

   - PGRST116 / 406 → "Không tìm thấy tuyến đường."
   - 401/403/PGRST301 → "Phiên đăng nhập đã hết hạn. Vui lòng đăng nhập lại."
   - Default → "Không thể tải tuyến đường. Vui lòng thử lại."

3. **Component structure:**
   - Mode detection: `id ? 'edit' : 'create'`
   - Edit mode: `useRoute(id)` with loading skeleton and error state
   - FK dropdowns: `useStations` for origin and destination (same cache key, deduplication)
   - FK dropdown empty states: when `originStations.length === 0` or `destStations.length === 0`, show "Chưa có trạm nào — tạo trạm trước ở /stations" and disable submit button (audit-added: prevents 23505 FK violation)
   - FK dropdown truncation warning: when `originCount > originStations.length` or `destCount > destStations.length`, show warning with AlertTriangle icon (audit-added: follows 10-02/10-03/10-04 pattern)
   - FormSection grouping:
     - Left column: "Thông tin chung" — name, origin station, destination station
     - Right column: "Chi tiết" — distance_km, estimated_duration_minutes (2-col grid), base_price, is_active switch
   - useBlocker with pathname guard: `isDirty && !isPending && currentLocation.pathname !== nextLocation.pathname`
   - hasInitializedRef for edit mode: prevents background refetch from overwriting edits
   - saveAndStops state: `const [saveAndStops, setSaveAndStops] = useState(false)` — toggled by "Lưu & Điểm dừng" button onClick (audit-added: submit handler references this)
   - "Lưu & Điểm dừng" button in create mode (like "Lưu & Phân công" in TripFormPage)
   - Sticky footer with Hủy + Lưu & Điểm dừng + Thêm/Lưu buttons
   - Dirty-state confirmation dialog

4. **Submit handler:**
   - Create: save → if saveAndStops, reset(values) then navigate to ROUTES_STOPS.replace(':id', result.id); else reset(values) then navigate to ROUTES.ROUTES
   - Edit: save → reset(values) then navigate to ROUTES.ROUTES
   - Error: mapSupabaseError with context 'mutate'

5. **Layout:** Full-page layout matching TripFormPage — flex flex-col h-full, page header with ArrowLeft back button, form with scrollable content + sticky footer.
