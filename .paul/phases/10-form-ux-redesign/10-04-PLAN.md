```gherkin

  - "Phân quyền" (Vai trò)
```

## AC-2: Full-page employee form — edit mode
```gherkin
Given an employee exists in the system
When the user clicks "Sửa" from the employees list dropdown
Then they navigate to /employees/{id}/edit
And the form is pre-filled with employee data and current role
And the page header shows "Chỉnh sửa nhân viên"

```

## AC-3: Two-step save with partial failure handling
```gherkin
Given the user submits the employee form
When the employee record saves successfully but role assignment fails
Then a warning toast appears: "Nhân viên đã được lưu, nhưng không thể cập nhật vai trò"
And the user is redirected back to the employees list
```

## AC-4: Dirty state guard
```gherkin
Given the user has modified form fields (isDirty = true)
When they navigate away (back button, sidebar link, browser back)
Then a confirmation dialog appears: "Thoát mà không lưu?"
And they can choose "Ở lại" or "Thoát"
And the guard does NOT trigger during submit (isPending) or same-pathname navigation
```

## AC-5: Edit mode loading and error states
```gherkin
Given the user navigates to /employees/{id}/edit
When the employee data is loading
Then a skeleton placeholder matching the form layout is shown
When the employee does not exist (PGRST116) or session expired (401/403/PGRST301)
Then an error state with Vietnamese message and "Quay lại danh sách" button is shown
```

```gherkin
When they click "Thêm nhân viên"
```

  <action>
Create `src/pages/employees/ui/employee-form-page.tsx` following the MaintenanceFormPage pattern exactly.

**Layout structure:**
- `<div className="flex flex-col h-full">` — root container
- `flex-none pb-4 border-b` — page header with ArrowLeft button + title
- `<form className="flex flex-col flex-1 min-h-0">` — form wrapper
  - `flex-1 overflow-y-auto` — scrollable content area with `py-6 px-1 space-y-8`
  - `flex-none border-t bg-background py-4 px-1` — sticky footer with Hủy/Submit buttons
- Dirty-state blocker dialog (same as MaintenanceFormPage)

**Mode detection:**
- `const { id } = useParams<{ id: string }>()`
- `const mode = id ? 'edit' : 'create'`
- Fetch employee data via `useEmployee(id ?? '')` for edit mode
- `hasInitializedRef` pattern for pre-filling edit form

**Form sections using FormSection component:**
1. "Thông tin chung" — Người dùng (profiles dropdown via Controller), Ngày vào làm (date input), Trạng thái hoạt động (Switch with Label)
2. "Bằng lái xe" — Số bằng lái (text input), Ngày hết hạn bằng lái (date input)
3. "Phân quyền" — Vai trò (roles dropdown via Controller)

**Two-column layout at section level:**
- Left: "Thông tin chung" and "Phân quyền" stacked vertically
- Right: "Bằng lái xe"
- `grid grid-cols-1 lg:grid-cols-2 gap-8 items-start`

**Form fields — replicate from existing employee-form-dialog.tsx:**
- `user_id`: Controller + Select with `__none__` sentinel (Radix pattern), profiles from `useProfiles()`, truncation warning when `profiles.length >= 1000`
  - **Empty state handling:** If profiles array is empty or isLoading, show "Chưa có người dùng — tạo người dùng trước" in SelectContent and disable submit
- `hire_date`: date input, nullable
- `license_number`: text input, max 50 chars, nullable
- `license_expiry`: date input, nullable
- `is_active`: Controller + Switch, default true
- `role_id`: Controller + Select with `__none__` sentinel, roles from `useRoles({ page: 1, pageSize: 1000 })`
  - **Empty state handling:** If roles array is empty or isLoading, show "Chưa có vai trò — tạo vai trò trước" in SelectContent and disable submit

**Role loading for edit mode:**
- `useEmployeeRole(employee?.user_id ?? null)` — wait until `currentRole !== undefined` before resetting form
- Only reset when both `log` (employee data) AND `currentRole` are loaded

**Submit logic (two-step save from dialog):**
1. Build payload: `{ user_id, hire_date, license_number, license_expiry, is_active }`
2. Step 1: `createMutation.mutateAsync(payload)` or `updateMutation.mutateAsync({ id, input: payload })`
3. On step 1 success, get `savedUserId` from result
4. Step 2: if `savedUserId`, call `assignEmployeeRole(savedUserId, values.role_id ?? null)`
5. On step 2 failure: warn toast + redirect (employee already saved)
6. On full success: success toast + `reset(values)` + `navigate(ROUTES.EMPLOYEES)`

**Error handling:**
- `mapFetchError` function for edit-mode fetch errors: PGRST116→"Không tìm thấy hồ sơ nhân viên.", 401/403/PGRST301→auth-expiry (add to employee-form-schema.ts or inline in page)
- Submit errors via `mapEmployeeError` from schema file
- Auth-expiry handling for two-step save: role assignment can fail with 401/403/PGRST301 — catch and handle gracefully (warn user, employee already saved)
- Submit error displayed in scrollable content area above footer (not in toast, to keep form open for correction) — render `{submitError && <p className="text-sm text-destructive">{submitError}</p>}` before the closing `</div>` of scrollable content, before footer

**Loading/error skeletons for edit mode:**
- Skeleton matching 2-column layout
- Error state with AlertCircle + Vietnamese message + back button

**useBlocker:**
```tsx
const blocker = useBlocker(
  ({ currentLocation, nextLocation }) =>
    isDirty && !isPending && currentLocation.pathname !== nextLocation.pathname
);
```
