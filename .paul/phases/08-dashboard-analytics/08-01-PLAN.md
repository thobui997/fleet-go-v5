## Goal
Build the dashboard landing page with KPI stat cards (vehicle fleet, trips today, bookings today, revenue today) and two quick-view tables (recent bookings, upcoming trips), replacing the existing placeholder.

## Purpose
The dashboard is the first page users see after login. It provides an at-a-glance operational overview — fleet health, today's activity, recent bookings, and upcoming trips — so operators can immediately assess the state of business without navigating to individual modules.

## Output
- Dashboard page with stat cards and quick views, wired as the app's landing route
- Dashboard-specific API layer with Supabase aggregation queries and TanStack Query hooks
</objective>

    1. **src/pages/dashboard/model/types.ts** — Dashboard-specific types:
       - `DashboardStats`: interface with fields: vehicleTotal (number), vehicleActive (number), tripsToday (number), tripsScheduled (number), tripsInProgress (number), bookingsToday (number), revenueToday (number)
       - `RecentBooking`: pick from BookingWithDetails — id, booking_code, customer full_name, route origin/destination names, trip departure_time, total_amount, status
       - `UpcomingTrip`: pick from TripWithDetails — id, route origin/destination names, departure_time, vehicle license_plate, status

    2. **src/pages/dashboard/api/dashboard.api.ts** — Aggregation queries using Supabase client:
       - `fetchDashboardStats()`: Run parallel queries using **Promise.allSettled** (not Promise.all) for resilience — if one query fails, others still return data. Structure the result to include per-query error state:
         ```typescript
         // Use Promise.allSettled so partial failures don't kill the entire dashboard
         const [vehiclesResult, tripsResult, bookingsResult, revenueResult] = await Promise.allSettled([...]);
         // Process each result: if 'fulfilled', extract data; if 'rejected', set defaults (0 values)
         ```
         - vehicles: `select('status', { count: 'exact' })` with no range (get all for status breakdown)
         - trips today: `select('*', { count: 'exact' }).gte('departure_time', todayStart).lte('departure_time', todayEnd)` — count total, compute status breakdown from data
         - bookings today: `select('*', { count: 'exact' }).gte('booking_date', todayStart).lte('booking_date', todayEnd)` — count
         - revenue today: `select('amount').eq('status', 'completed').gte('paid_at', todayStart).lte('paid_at', todayEnd)` — sum amounts client-side
         Use dayjs for today's date boundaries: `dayjs().startOf('day').toISOString()` and `dayjs().endOf('day').toISOString()`

       - `fetchRecentBookings()`: `select(BOOKING_SELECT, { count: 'exact' }).order('booking_date', { ascending: false }).limit(5)` — reuse BOOKING_SELECT from booking.api.ts (copy the select string, do NOT import from booking entity to avoid cross-import)
       - `fetchUpcomingTrips()`: `select(TRIP_SELECT, { count: 'exact' }).gte('departure_time', now).order('departure_time', { ascending: true }).limit(5)` — reuse TRIP_SELECT from trip.api.ts (copy the select string)

    3. **src/pages/dashboard/api/dashboard.queries.ts** — TanStack Query hooks:
       - `useDashboardStats()`: useQuery with queryKey ['dashboard', 'stats'], staleTime 30_000 (30 seconds)
       - `useRecentBookings()`: useQuery with queryKey ['dashboard', 'recent-bookings']
       - `useUpcomingTrips()`: useQuery with queryKey ['dashboard', 'upcoming-trips']
       - Auth-expiry handling: In each query's onError or in the API functions, detect 401/403/PGRST301 errors. On detection, call `supabase.auth.signOut()` to force re-login (consistent with all prior plan patterns). Add a helper function:
         ```typescript
         function handleAuthExpiry(error: unknown) {
           const msg = error instanceof Error ? error.message : '';
           const code = (error as { code?: string })?.code;
           if (code === 'PGRST301' || msg.includes('401') || msg.includes('403')) {
             supabase.auth.signOut();
           }
         }
         ```
         Call this in each API function's catch block or use TanStack Query's `onError` callback. <!-- audit-added -->

    4. **src/pages/dashboard/index.ts** — Public API:
       ```
       export { DashboardPage } from './ui/dashboard-page';
       ```

    IMPORTANT:
    - Do NOT import from @entities/booking or @entities/trip — copy the SELECT strings to avoid FSD cross-import violations
    - Use `import { supabase } from '@shared/api'` for Supabase client
    - Use `import dayjs from 'dayjs'` for date calculations
    - All errors should be thrown directly (Supabase error), no custom error mapping needed for dashboard
    - **fetchDashboardStats must use Promise.allSettled** (not Promise.all) — prevents one failing query from breaking all stat cards <!-- audit-added -->
    - Auth-expiry detection: handle PGRST301, 401, 403 by signing user out (same pattern as all prior plans) <!-- audit-added -->
