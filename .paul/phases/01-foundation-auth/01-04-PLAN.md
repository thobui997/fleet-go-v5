       <!-- audit-added: Dark mode flash prevention -->
       **Dark mode flash (FOUC) prevention:**
       The header component's useEffect for reading localStorage theme runs AFTER React hydrates, causing a flash of wrong theme on page load. To prevent this, add an inline `<script>` block in `index.html` BEFORE the CSS `<link>`:

       ```html
       <script>
         try {
           const theme = localStorage.getItem('theme');
           if (theme === 'dark' || (!theme && window.matchMedia('(prefers-color-scheme: dark)').matches)) {
             document.documentElement.classList.add('dark');
           }
         } catch (_) {}
       </script>
       ```

       Place this in `index.html` inside `<head>`, before any stylesheet links. The header component still manages the toggle and localStorage persistence — this script only ensures the correct class is set synchronously before first paint.

       Dark mode toggle implementation in header:
       - Read current state from `document.documentElement.classList.contains('dark')` for initial React state
       - Toggle adds/removes 'dark' class on `<html>` element
       - Persist choice to localStorage key 'theme' ('dark' or 'light')
       - Sync state with the DOM on mount via useEffect

       <!-- audit-added: Logout error handling -->
       **Logout error handling:**
       The logout button calls `useAuth().logout()` which invokes `supabase.auth.signOut()`. If this network call fails (session expired, network error, Supabase unreachable), the promise rejects silently and the user sees no feedback. Wrap the logout handler in a try/catch:
       - On success: redirect to /login (existing behavior)
       - On error: show a toast notification ("Đăng xuất thất bại. Vui lòng thử lại.") and log the error. Still attempt to redirect to /login — the local session may be invalid anyway, so forcing navigation is safer than leaving the user stuck.
