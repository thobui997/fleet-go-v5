    1. Create `src/shared/ui/data-table.tsx` — a generic, reusable DataTable component:

       ```typescript
       Design requirements:
       - Generic TypeScript component: DataTable<TData, TValue>
       - Props: columns (ColumnDef[]), data (TData[]), pagination (page, pageSize, total, onPageChange, onPageSizeChange), isLoading?, emptyMessage?
       - Uses the Shadcn/ui Table component for styling
       - Column headers are clickable for sorting (asc/desc toggle with chevron icon from lucide-react)
       - Pagination controls at the bottom:
         - "Previous" / "Next" buttons (disabled at boundaries)
         - Page number display: "Page X of Y"
         - Page size selector: dropdown with options [10, 20, 50, 100]
       - Loading state: renders Skeleton rows when isLoading is true
       - Empty state: renders emptyMessage text centered when data array is empty
       - Responsive: horizontal scroll on narrow screens
       ```

       The component should NOT depend on TanStack Table (to keep dependencies minimal). Instead, use simple state-based sorting and pagination passed as props. The parent component handles the actual data fetching via TanStack Query.

       <!-- audit-added: Column interface contract -->
       **ColumnDef interface (defined at the top of data-table.tsx):**
       ```typescript
       export interface ColumnDef<TData> {
         key: keyof TData & string;
         header: string;
         cell?: (value: TData[keyof TData], row: TData) => React.ReactNode;
         sortable?: boolean;
         className?: string;
       }

       export interface DataTablePagination {
         page: number;
         pageSize: number;
         total: number;
         onPageChange: (page: number) => void;
         onPageSizeChange: (size: number) => void;
       }

       export interface DataTableProps<TData> {
         columns: ColumnDef<TData>[];
         data: TData[];
         pagination: DataTablePagination;
         isLoading?: boolean;
         emptyMessage?: string;
       }
       ```
       This interface is a shared contract used across all phases. Future consumers (vehicles, routes, employees, trips, etc.) will import ColumnDef and DataTablePagination to define their table configurations.

    Avoid: Adding TanStack Table as a dependency (unnecessary for this use case). Making the DataTable tightly coupled to any specific data fetching pattern. Adding complex features like row selection or inline editing (those come per-feature, not shared).
