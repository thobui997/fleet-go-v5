<objective>
## Goal
Build Routes CRUD — entity slice (`@entities/route`) + list page (name search + is_active filter) + create/edit form dialog (two station FK dropdowns, interval duration, distance, price) + delete dialog + router wiring.

## Purpose
Routes are the foundation for trip scheduling (Phase 6). They link two stations with travel metrics. This plan makes routes visible and manageable in the back-office before 04-03 adds intermediate stops.

## Output
- `@entities/route` public API: `Route`, `RouteInsert`, `RouteUpdate`, `RouteListParams`, `useRoutes`, `useRoute`, `useCreateRoute`, `useUpdateRoute`, `useDeleteRoute`
- `/routes` page replacing PlaceholderPage with full CRUD
- Station name joins in list view (PostgREST FK join)
- `estimated_duration` handled as minutes in form, serialized to PostgreSQL interval format
</objective>

    `getRoutes({ page, pageSize, search, isActive })`:

    `createRoute(input: RouteInsert)`: `.from('routes').insert(input).select().single()`

    `updateRoute(id, input: RouteUpdate)`: `.from('routes').update(input).eq('id', id).select().single()`

    `deleteRoute(id)`: `.from('routes').delete().eq('id', id)`

    Throw the Supabase error on failure (check error !== null after each call).

    Zod schema (`routeFormSchema`):
    ```ts
    z.object({
      name: z.string().trim().min(1, 'Tên tuyến đường không được để trống'),
      origin_station_id: z.string().min(1, 'Vui lòng chọn trạm đi'),
      destination_station_id: z.string().min(1, 'Vui lòng chọn trạm đến'),
      distance_km: z.coerce.number().positive('Khoảng cách phải lớn hơn 0'),
      estimated_duration_minutes: z.coerce.number().int().positive('Thời gian dự kiến phải lớn hơn 0'),
      base_price: z.coerce.number().min(0, 'Giá vé không được âm'),
      is_active: z.boolean().default(true),
    }).superRefine((data, ctx) => {
      if (
        data.origin_station_id &&
        data.destination_station_id &&
        data.origin_station_id === data.destination_station_id
      ) {
        ctx.addIssue({
          code: z.ZodIssueCode.custom,
          message: 'Trạm đi và trạm đến không được trùng nhau',
          path: ['destination_station_id'],
        });
      }
    })
    ```

    `mapSupabaseError(error: { code?: string; message?: string; details?: string; status?: number }, context?: 'mutate' | 'delete')`: <!-- audit-added: context param distinguishes INSERT/UPDATE 23503 from DELETE 23503 -->
    - 401/403/PGRST301 → auth expiry message
    - 23505 + (`routes_name_key` in message OR `(name)` in details) → "Tên tuyến đường đã tồn tại"
    - 23503 + context==='delete' (or no context) → "Không thể xóa tuyến đường đang được sử dụng bởi chuyến đi"
    - 23503 + context==='mutate' → "Trạm không tồn tại hoặc đã bị xóa" <!-- audit-added: INSERT FK violation on station race condition -->
    - 23514 → "Dữ liệu không hợp lệ (vi phạm ràng buộc kiểm tra)" <!-- audit-added: matches station-form-schema.ts; catches CHECK violations from DB -->
    - default → "Thao tác thất bại. Vui lòng thử lại."
    
    Call sites:
    - form dialog `catch`: `mapSupabaseError(error, 'mutate')`
    - delete dialog `catch`: `mapSupabaseError(error, 'delete')` or `mapSupabaseError(error)` <!-- audit-added -->

    `serializeToInsert(values: RouteFormValues): RouteInsert`:
    - Convert `estimated_duration_minutes` to interval string:
      ```ts
      const safeMinutes = Math.max(1, values.estimated_duration_minutes); // audit-added: clamp to min 1 — prevents "00:00:00" reaching DB CHECK
      const hours = Math.floor(safeMinutes / 60);
      const mins = safeMinutes % 60;
      const estimated_duration = `${String(hours).padStart(2,'0')}:${String(mins).padStart(2,'0')}:00`;
      ```
    - Map all other fields directly (distance_km, base_price are numbers from z.coerce)

    `parseDurationMinutes(interval: string): number` — helper for edit mode: <!-- audit-added: robust multi-format parser -->
    - Handles two PostgreSQL interval formats returned by Supabase/PostgREST:
      1. Standard: "HH:MM:SS" → split by ':', parse hours * 60 + minutes
      2. Multi-day: "X days HH:MM:SS" → split on ' days ', parse day portion * 1440 + HH * 60 + MM
    - Also handle "H:MM:SS" (no leading zero on hours) — use parseInt() not fixed-width parse
    - Falls back to `1` (minimum valid) if parsing fails, NOT 0 — returning 0 would submit "00:00:00" which violates the DB CHECK constraint `routes_duration_positive`
    - Example: "02:30:00" → 150; "1 days 00:30:00" → 1470; unparseable → 1
