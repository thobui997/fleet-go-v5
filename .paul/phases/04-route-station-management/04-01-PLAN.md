```gherkin
Given the user navigates to /stations

     and a "Thêm trạm" button in the page header
```

## AC-2: Search filter
```gherkin
Given the stations list page
When the user types in the search input (300ms debounce)
Then the DataTable filters to stations whose name OR city matches the search term (ilike)
     and the page resets to 1
```

## AC-3: Status filter
```gherkin
Given the stations list page
When the user selects "Hoạt động" / "Ngừng hoạt động" / "Tất cả" from the status Select
Then the DataTable shows only stations matching that is_active value
     and the page resets to 1
```

## AC-4: Create station
```gherkin
Given the user clicks "Thêm trạm"
When the form dialog opens, they fill in required fields (name, city) plus optional fields, then submit
Then the station is created in Supabase,
     the stations list invalidates and refreshes,
     and a success toast shows "Đã tạo trạm"
```

## AC-5: Edit station
```gherkin
Given the user clicks the edit action on a station row
When the form dialog opens pre-filled with the station's current values
     and the user modifies one or more fields and submits
Then the station is updated,
     the list refreshes,
     and a success toast shows "Đã cập nhật trạm"
```

## AC-6: Delete station
```gherkin
Given the user clicks the delete action on a station row
When the delete confirm dialog opens and the user clicks "Xóa"
Then the station is deleted,
     the list refreshes,
     and a success toast shows "Đã xóa trạm"
```

## AC-7: Form validation (Zod, Vietnamese messages)
```gherkin
Given the create/edit form dialog is open
When the user submits with invalid or missing values
Then Zod validation errors display inline in Vietnamese:
     - name empty/whitespace → "Tên trạm không được để trống"
     - city empty/whitespace → "Thành phố không được để trống"
     - latitude not in [-90, 90] → "Vĩ độ phải trong khoảng -90 đến 90"
     - longitude not in [-180, 180] → "Kinh độ phải trong khoảng -180 đến 180"
     - code > 20 chars → "Mã trạm quá dài (tối đa 20 ký tự)"
     (no submission to Supabase until validation passes)
```

## AC-8: Supabase error mapping (Vietnamese)
```gherkin
Given a mutation or query returns a Supabase error
When mapSupabaseError processes the error
Then it returns the correct Vietnamese message:
     - 23505 constraint "stations_name_key" (check message includes 'stations_name_key' OR details includes '(name)') → "Tên trạm đã tồn tại"
     - 23505 constraint "stations_code_key" (check message includes 'stations_code_key' OR details includes '(code)') → "Mã trạm đã tồn tại"
     - 23503 on delete (RESTRICT from routes/route_stops) → "Không thể xóa trạm đang được sử dụng bởi tuyến đường"
     - 23514 CHECK violation → "Dữ liệu không hợp lệ (vi phạm ràng buộc kiểm tra)"
     - 401 / 403 / PGRST301 → "Phiên đăng nhập đã hết hạn hoặc bạn không có quyền. Vui lòng đăng nhập lại."
     - fallback → "Thao tác thất bại. Vui lòng thử lại."
```

## AC-9: List error state with retry
```gherkin
Given the fetchStations query returns an error
When the station list page renders
Then an inline AlertCircle error message displays
     with a "Thử lại" button that calls refetch()
     (not an empty or broken DataTable)
```

## AC-10: Dialog close guard during pending mutation
```gherkin
Given a form or delete dialog is open and a mutation is in-flight (isPending = true)
When the user clicks outside the dialog or presses Escape
Then the dialog ignores the close event and stays open
     until the mutation resolves or rejects
```

    **station-form-schema.ts:**
    - `stationFormSchema` (Zod):
      - `name`: z.string().trim().min(1, 'Tên trạm không được để trống')
      - `code`: z.string().trim().max(20, 'Mã trạm quá dài (tối đa 20 ký tự)').optional().or(z.literal(''))
      - `address`: z.string().optional().or(z.literal(''))
      - `city`: z.string().trim().min(1, 'Thành phố không được để trống')
      - `province`: z.string().optional().or(z.literal(''))
      - `latitude`: z.union([z.coerce.number().min(-90, 'Vĩ độ phải trong khoảng -90 đến 90').max(90, 'Vĩ độ phải trong khoảng -90 đến 90'), z.literal('')]).optional()
      - `longitude`: z.union([z.coerce.number().min(-180, 'Kinh độ phải trong khoảng -180 đến 180').max(180, 'Kinh độ phải trong khoảng -180 đến 180'), z.literal('')]).optional()
      - `is_active`: z.boolean().default(true)
      - IMPORTANT: latitude and longitude Input components MUST use `type="text"` (NOT `type="number"`) — z.coerce.number() handles string→number coercion; `type="number"` returns NaN on partial input (e.g., typing "-" before completing "-90") which breaks the z.literal('') union branch <!-- audit-added: type="text" is the correct pattern for z.union([z.coerce.number(), z.literal('')]) fields -->
    - `mapSupabaseError(error: { code?: string; message?: string; details?: string; status?: number })`: <!-- audit-added: include details field for constraint-specific 23505 discrimination -->
      - 401/403/PGRST301 → auth expiry message
      - 23505 + (message includes 'stations_name_key' OR details includes '(name)') → "Tên trạm đã tồn tại" <!-- audit-added: specific check; generic includes('name') would match unrelated error contexts -->
      - 23505 + (message includes 'stations_code_key' OR details includes '(code)') → "Mã trạm đã tồn tại" <!-- audit-added: specific constraint check -->
      - 23505 fallback → "Giá trị đã tồn tại"
      - 23503 → "Không thể xóa trạm đang được sử dụng bởi tuyến đường"
      - 23514 → "Dữ liệu không hợp lệ (vi phạm ràng buộc kiểm tra)"
      - default → "Thao tác thất bại. Vui lòng thử lại."
    - `serializeToInsert(values): StationInsert`:
      - name: values.name (already trimmed by Zod)
      - code: !values.code || values.code.trim() === '' ? null : values.code.trim()
      - address: !values.address || values.address.trim() === '' ? null : values.address
      - city: values.city
      - province: !values.province || values.province.trim() === '' ? null : values.province
      - latitude: values.latitude === '' || values.latitude === undefined ? null : Number(values.latitude)
      - longitude: values.longitude === '' || values.longitude === undefined ? null : Number(values.longitude)
      - is_active: values.is_active ?? true
