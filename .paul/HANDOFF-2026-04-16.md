### Auto-Fix Discovered

**PostgREST PGRST201 Relationship Ambiguity**
- **Issue:** Bookings API returned "Could not embed because more than one relationship was found for 'bookings' and 'tickets'"
- **Root cause:** Composite FK `(booking_id, trip_id)` created ambiguity — PostgREST couldn't determine which FK to use
- **Fix:** Added explicit FK constraint name `!tickets_booking_id_fkey` to nested selects:
  - `BOOKING_WITH_TICKETS_SELECT` in `src/entities/booking/api/booking.api.ts`
  - `fetchTicketsByBookingCode` in `src/entities/ticket/api/ticket.api.ts`
