## Key Technical Decisions

| Decision | Rationale |
|----------|-----------|
| phone_number trimmed in serializeToInsert | Prevents UNIQUE constraint bypass (e.g., " 0901..." vs "0901...") |
| email/id_card_number empty→null | PostgreSQL allows multiple NULLs but rejects multiple empty strings for UNIQUE nullable columns |
| Gender Select uses DB values directly | Options: 'male'/'female'/'other' — no label→value mapping needed in serialization |
| hasInitializedRef pattern | Prevents background refetch from overwriting form fields during edit mode |
| Auth-expiry error mapping (401/403/PGRST301) | Consistent Vietnamese session expiry message across all pages |
